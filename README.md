Задание 1. Описание тематики дипломного проекта

Актуальность темы
Многие частные мастера, самозанятые и индивидуальные предприниматели (ремонтники,
мастера по изготовлению изделий, специалисты сферы красоты и другие) ведут учёт своих
заказов в бумажных блокнотах, заметках телефона или переписке в мессенджерах. Такой
подход приводит к ряду проблем:
· потеря контактов клиентов;
· забытые суммы предоплат и остатков долга;
· отсутствие прозрачной картины дохода за месяц;
· путаница со статусами заказов (что сделано, а что ещё нет);
· невозможность быстро найти нужный заказ или клиента.
Существующие CRM-системы зачастую перегружены ненужным функционалом, требуют
платной подписки или слишком сложны для освоения человеком без технического
образования.
В связи с этим разработка простого, понятного и адаптированного под нужды конкретного
мастера веб-приложения является актуальной задачей. Такое приложение поможет
мастеру экономить время, не терять данные и иметь полный контроль над своими
заказами и финансами.
Целевая аудитория
· Частные мастера услуг (ремонт техники, изготовление изделий на заказ, швейные услуги,
ювелирные работы и т.д.);
· Индивидуальные предприниматели, работающие без штата сотрудников;
· Самозанятые граждане, которым нужен минимальный, но достаточный учёт;
· В перспективе — мастера, желающие оцифровать свой учёт без перехода на дорогие
CRM.
Предполагаемые функциональные возможности
1. Управление клиентами

 Добавление, просмотр, редактирование и удаление клиентов (имя, номер телефона,
комментарий).
2. Управление заказами

 Создание заказа с привязкой к клиенту, описанием работы, датой приёма, полной
стоимостью и предоплатой.
 Автоматический расчёт остатка долга.
3. Статусы заказов

 Возможность менять статус заказа: «Новый», «В работе», «Готов к выдаче», «Выдан».
4. Учёт платежей

 Возможность вносить частичные оплаты по заказу. Автоматическое обновление остатка
долга.
5. Поиск и фильтрация

 Поиск заказов по имени клиента или номеру телефона.
 Фильтрация заказов по статусу.
6. Просмотр должников

 Отдельный список заказов, по которым остаток долга больше нуля.
7. Простая статистика

 Количество заказов в работе, количество готовых заказов, общая сумма долгов, выручка
за выбранный период.
Используемые технологии (предварительно)
Компонент Технологии

Frontend (клиентская часть) HTML5, CSS3, JavaScript
Backend (серверная часть) Node.js + Express (или PHP)
База данных MySQL / PostgreSQL / SQLite
Среда разработки VS Code
Контроль версий (опционально) Git
Ожидаемый результат

Работающее веб-приложение, которое:
· позволяет мастеру вести полный учёт клиентов и заказов;
· хранит все данные в базе данных;
· предоставляет понятный веб-интерфейс (работает через браузер);
· даёт возможность отслеживать статусы заказов, долги клиентов и доходы;
· готово к использованию в реальной профессиональной деятельности мастера.
Дополнительно

Данный проект может стать основой для дипломной работы и в дальнейшем быть
расширен следующими функциями:
· авторизация и работа сразу нескольких мастеров;
· отправка напоминаний клиентам через SMS или Telegram;
· экспорт отчётов в Excel/PDF;
· мобильная адаптация для использования со смартфона
